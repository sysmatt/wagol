# conweba

Conway's Game of Life running in WebAssembly, doubling as a "site under construction" holding page. The simulation fills the entire browser window with each cell rendered as a single pixel; an optional image and text can be overlaid, centered on top of it.

## Demo

Open `http://localhost:8080` after running `make serve`.

## How it works

The simulation core is written in Rust and compiled to WebAssembly via `wasm-pack`. Each tick, the Wasm module computes the next generation using a double-buffered cell array and writes the result directly into a pre-allocated RGBA pixel buffer. JavaScript creates an `ImageData` view directly into Wasm linear memory (zero-copy) and blits it to the canvas each frame via `putImageData`.

- Grid dimensions match the browser window exactly — one pixel per cell
- Edges wrap around (toroidal topology)
- Grid is re-randomized (~50% alive) on load and on window resize
- Render loop runs at full `requestAnimationFrame` speed
- Cell color is driven by cell age and a selectable palette (see Config below)
- Dragging the mouse scatters a sparse, randomized burst of live cells rather than painting a solid blob

## Config

Holding-page content and behavior are set via the `CONFIG` object at the top of `index.js`:

```js
const CONFIG = {
    theme: 'cosmic', // 'cosmic' | 'fire' | 'matrix'
    brush: { radius: 35, density: 0.35 },
    image: { src: '', opacity: 0.35, maxWidth: 320, maxHeightVh: 40 },
    text: { content: '', color: '#ffffff', fontSize: '1.5rem' },
};
```

Leave `image.src` or `text.content` empty to omit that layer. Both render as a centered, `pointer-events: none` overlay above the canvas, so mouse interaction always reaches the simulation.

## Requirements

On Ubuntu:

```bash
sudo apt install rustup binaryen build-essential
rustup target add wasm32-unknown-unknown
cargo install wasm-pack
```

- `rustup` — manages the Rust toolchain; `binaryen` supplies `wasm-opt` (used by wasm-pack to optimize the `.wasm` output, so it doesn't need to fetch its own copy); `build-essential` provides the native `cc`/linker that `cargo install wasm-pack` and other host-side build scripts need
- `wasm32-unknown-unknown` — the actual GoL simulation compiles to this target and needs no C toolchain itself; only the *build tools* (wasm-pack) are native binaries
- Python 3 (for the dev server) — ships with Ubuntu

`make build` checks these are present and gives a hint if something's missing.

## Build and run

```bash
make serve       # build and start server at http://localhost:8080
make build       # build only (output in pkg/)
make tar.gz      # build a deployable conweba.tar.gz (html/js/wasm/images only)
make clean       # remove build artifacts
```

## Wasm API

The Rust library exposes a single `Universe` struct:

| Method | Description |
|---|---|
| `Universe.new(width, height, theme)` | Create and randomly seed a new universe. `theme` is `0` (fire), `1` (cosmic), or `2` (matrix) |
| `universe.tick()` | Advance one generation |
| `universe.pixels_ptr()` | Pointer into Wasm memory for the RGBA pixel buffer |
| `universe.scatter_cells(x, y, radius, density)` | Randomly scatter live cells within `radius` of a point, with density (0–1) falling off toward the edge |
| `universe.resize(width, height)` | Reinitialize with new dimensions, keeping the current theme |
| `wasm_memory()` | Returns the `WebAssembly.Memory` object for direct buffer access |
